# Contributing

Thank you for your interest in contributing to this project.

---

## Contribution goals

This project aims to improve:

* structure and consistency of AI-generated code
* maintainability of application codebases
* clarity of technical documentation
* long-term evolution of AI-assisted development practices

All contributions should align with these goals.

---

## Before contributing

Please review:

* `README.md`
* `docs/philosophy.md`
* `docs/how-it-works.md`
* `VERSIONING.md`

Understanding the intent of the project is important before proposing changes.

---

## Types of contributions

We welcome:

* improvements to clarity and wording
* improvements to consistency
* additions to templates that increase practical value
* improvements to examples
* schema enhancements
* documentation improvements

---

## Contributions requiring extra care

Changes affecting the following require strong justification:

* template structure
* section names or ordering
* required fields
* rules and validation logic
* skill behavior

These changes impact all downstream users.

---

## Guidelines

* Keep the structure stable
* Prefer additive changes over breaking changes
* Avoid unnecessary complexity
* Keep content explicit and structured
* Avoid vague or interpretative wording

---

## Template changes

If proposing changes to the template:

* explain why the change improves real-world usage
* explain how it impacts existing documents
* indicate whether it is backward compatible

---

## Skill changes

If modifying a skill:

* keep behavior predictable
* avoid introducing ambiguity
* ensure alignment with existing rules and checklist

---

## Pull requests

When submitting a pull request, include:

* what changed
* why it is needed
* whether the change is additive or breaking
* which parts of the system are affected

---

## Discussions

For significant changes, open a discussion before submitting a pull request.

---

## General principle

This project values:

* consistency over flexibility
* clarity over cleverness
* structure over freedom

The goal is to build a standard that remains reliable over time.
